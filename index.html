Summary

React is a free and open-source JavaScript library that can be used to create applications for web, mobile and desktop. Developed and released by Facebook in May of 2013, React has grown to be the single most popular and widely used library out of its competitors and offers a wide array of tools to aid developers in building their projects.

Skills and prerequisites

Built around JavaScript and with primary focus on web applications, getting started with React requires good understanding of HTML, CSS, JavaScript itself especially, the Command Line and use/installation of Node/Node Package Manager.

Implementation

Node package manager is what you’ll use to install your React Application as well as any extra additions (like Bootstrap) within it, only needing to run the "npm install -g create-react-app" command in your Command Line to create a basic starting template. Once installation is finished, you’ll have a project folder with some basic structuring to start working in, only then needing to Command Line “cd” inside the new folder and enter “npm start” to run the application.

You might not need a library of this caliber for very small and simple projects, but it’ll save a lot of time and headache in helping manage larger ones with complicated and heavily interactable UI.

Pros and cons

    Pros
  • Being as popular as it is, React has massive community support and you’ll find lots of help troubleshooting and learning to use it.
  • React is very flexible and adaptable in the way it works, giving developers a lot of freedom in the way they want to utilize it.
  • The learning curve isn’t bad at all, and so long as you have decent understanding of programming, it’ll only take a few weeks at most to get things down.
  • React uses Virtual DOM to keep application rendering incredibly fast and efficient.
  • Great for UI creation through the use of custom and reusable components.
    Cons
  • The rapid development of React can lead to some issues for those trying to use it, as changes/additions to tools and the way code functions can leave older React Applications with compatibility issues and bugs. On top of this, proper and fully flushed out documentation can also be lacking at times as it takes some to write/update.
  • While the flexibility and open-endedness of React offers a lot of freedom, it can also be a double edge sword. Where one might already have a general idea of what to expect from more streamlined and convention heavy frameworks, the lack of such here may have React Developers that’re moving from one team to another needing to take extra time to learn new development processes/standards.
  • Unlike actual full fledged frameworks like Angular, React only touches on the UI portions of an application and nothing more, which may require developers to find and incorporate extra tools to fully flush things out.

Helpful tips

As previously mentioned, you can create a React Application using this Npm command in the Command Line.

npm install -g create-react-app

You can then start your React Application with this command

npm start

This command here is for compiling your application into a production build. In doing so it creates a new folder, automatically translating the code for browers through babel and compressing/minifying the project for performance and download efficiency

npm run build
